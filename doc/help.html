Configuration

At present, NETSPYRE provides configuration options with the help of the netspyre.conf file which is located within NETSPYRE's root folder. When NETSPYRE is launched, it would verify the presence of this file. If it is not found, NETSPYRE would instead generate a new one with default settings. However in the upcoming versions, the developer of NETSPYRE aspires to introduce a user-friendly settings menu. This enhancement will eliminate the need for manual file editing and offer a more convenient and secure way to modify the configurations.

CAUTION: Please refrain from deleting any of the lines within the netspyre.conf file, except for the tools, as it could result in a loss of functionality of the system. Additionaly, kindly be aware that the configuration values are case-sensitive.

 

Incorporating new actions

In NETSPYRE, it's actions are categorized into three distinct classes:

  • Host Actions: These actions are typically triggered by right-clicking on a host and then the results produced by them (the tools) are saved and shown within NETSPYRE.
  • Port Actions: It can initiate these actions by right-clicking on a specific port or service The tool output which is then geenrated would also store and be shown within NETSPYRE. Nikto is a great example of a port action.
  • Terminal Actions: When you right-click on a port this would trigger an external terminal window. For example, you would be able to open the FTP client via Terminal Actions.

 

To set up a new action, the following format should be adhered:

tool=label, command, services

tool is a unique identifier, typically the name of the tool.

label is the text that will appear in the context menu.

command field specifies the non-interactive terminal command which was used to execute the tool. It is also important to make sure that the command does not require manual input. During the runtime, parameter such as [IP], [PORT], and [OUTPUT] will be automatically substituted with the appropriate values.

services field holds a list of nmap service nmaes to which the tool is most applicable for. When you would right-click on a specific port of service, the tool will only appear in the meny if it is associated with a specific service. This field is not utilized in the Host Actions for obvious reasons.

 

Example:
To set up Nikto as a port action, certain strings should be included in the following line within the [PortActions] segment of the netspyre.conf file:

nikto=Run nikto, nikto -o [OUTPUT].txt -p [PORT] -h [IP], “http,https”

 

Once the configuration file has been modified, it's necessary to restart NETSPYRE to apply these changes. However, in the future, this step will no longer be applicable with the introduction of the settings menu.

 

Configuring automated attacks

NETSPYRE can also be configured to initiate the automatic execution of any sort of tool that has been defined with the [PortActions] segment of the configuration file when a service has been detected.

By default, the automated attacks are enabled. but they can easily be deactivated by modifying the "enable-scheduler" option which is found in the [GeneralSettings] section part of the configuration file.

The setup for automated attacks is specified in the [SchedulerSettings] section of the configuration file, using the following format:

tool=services, protocol

tool serves as the unique identifier that was previously employed to define the tool within the [PortActions] section.

services designates the protocol (tcp/udp) on which the tool should operate in response to the identified service.

protocol is the protocol of the service the tool should run on (tcp/udp).
